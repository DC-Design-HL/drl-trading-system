### TIF GTE Bug Fix
- Algo orders failed with `-4509` ("TIF GTE can only be used with open positions") 
- Root cause: `timeInForce` was being sent for market-type algo orders; Binance maps it to GTE internally
- Fix: removed `timeInForce` for STOP_MARKET/TAKE_PROFIT_MARKET, added 2s delay after position open, 3x retry logic

### SL/TP Update Bug Fix (-4130 conflict)
- `update_sl_tp()` was re-placing BOTH SL and TP even when only one changed → duplicate orders
- Error `-4130` ("An open stop or take profit order with GTE and closePosition in the direction is existing")
- Fix: only update changed orders (>$0.01 difference), clear stale algo orders on -4130 and retry
- Validated: cancel+replace does NOT affect entry price (6 tests, all passing on live exchange)

- Chart data source fixed: was reading from empty `state.get('trades')`, now fetches from `/api/testnet/trades`

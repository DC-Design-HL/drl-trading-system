🔍 PRIMARY ISSUE:
   System was started with --assets BTCUSDT only (missing 3 assets)
   
   Expected: python live_trading_multi.py --assets BTCUSDT ETHUSDT SOLUSDT XRPUSDT
   Actual:   python live_trading_multi.py --assets BTCUSDT
   
   Result: Only BTCUSDT bot exists in memory, state only tracks BTCUSDT

🔍 SECONDARY ISSUE:
   Bot realized_pnl not reconstructed from trade history on startup
   
   Code: self.realized_pnl = 0.0  # ❌ Always starts fresh
   
   Result: Even when bots exist, their P&L doesn't match historical trades

                               THE FIX
═══════════════════════════════════════════════════════════════════════════════

✅ CODE CHANGES (live_trading_multi.py):

   1. Enhanced load_state()
      → Auto-reconstructs missing asset P&L from trade history
      → Validates existing P&L against trade database
      → Self-heals on every startup

   2. New _reconstruct_bot_state()
      → Rebuilds bot state from trade database
      → Calculates total P&L from all historical trades
      → Recovers lost position info

   3. New _calculate_pnl_from_trades()
      → Source of truth: always query trade database
      → Used for validation and recovery

   4. Enhanced save_state()
      → Validates P&L every 5 minutes
      → Logs errors if discrepancies detected
      → Ongoing monitoring active
